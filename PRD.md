# COZY 주문 앱 PRD

## 1) 문서 개요
- 문서명: COZY 주문 앱 PRD
- 범위: `주문하기` 화면, `관리자` 화면
- 목적: 화면 UI/상태 전이/권한/재고 연동 규칙을 명확히 정의하여 개발 기준으로 사용

## 2) 공통 화면 구조
### 2.1 헤더
- 좌측에 브랜드명 `COZY` 표시
- 우측에 `주문하기`, `관리자` 내비게이션 제공
- 현재 화면 탭은 active 스타일로 구분

## 3) 주문하기 화면 PRD
### 3.1 목표
- 고객이 메뉴 선택, 옵션 선택, 장바구니 관리, 주문 제출을 한 화면에서 완료

### 3.2 메뉴 카드
- 카드 구성: 이미지, 메뉴명, 기본가격, 간단 설명, 옵션 체크박스, `담기` 버튼
- 옵션 예시: `샷 추가(+500원)`, `시럽 추가(+0원)`
- `담기` 클릭 시 선택 옵션 포함하여 장바구니 반영

### 3.3 장바구니
- 항목 정보: 메뉴명, 옵션, 수량, 소계 금액
- 하단에 총 금액 및 `주문하기` 버튼 표시

### 3.4 장바구니 규칙 (확정)
- 동일 메뉴 재담기 시:
  - 메뉴 + 옵션 조합이 동일하면 줄 병합
  - 병합 시 수량만 증가
  - 메뉴가 같아도 옵션 조합이 다르면 별도 줄 유지
- 장바구니 내 옵션 수정:
  - 이미 담은 항목의 옵션 해제/변경 불가
- 항목 삭제:
  - 각 줄 맨 끝 `X` 버튼으로만 삭제 가능
- 수량 변경:
  - 수량 옆 화살표 버튼(증가/감소) 제공
  - 수량이 1이면 감소 버튼 비활성화
  - 수량 1에서 감소 버튼으로 삭제 불가, 반드시 `X` 버튼으로 삭제
- 주문 실패:
  - 주문 실패 시 장바구니 유지

### 3.5 재고/품절 반영
- 재고가 0보다 크면 `담기` 버튼 노출
- 재고가 0이면 `담기` 버튼 숨김
- 같은 위치에 `품절` 텍스트 표시
- 재고/품절 고도화 정책은 추후 확장

## 4) 관리자 화면 PRD
### 4.1 목표
- 주문 상태를 단계적으로 처리하고, 재고를 즉시 조정하며, 대시보드 수치를 일관되게 유지

### 4.2 관리자 대시보드
- 지표 표시: `총 주문 / 주문 접수 / 제조 중 / 제조 완료`
- 주문 상태 변경 시 수치 동기화

### 4.3 재고 현황
- 메뉴별 재고 카드 표시(메뉴명, 수량, `+`, `-`)
- `+` 클릭: 재고 +1
- `-` 클릭: 재고 -1 (0 미만 불가)

### 4.4 주문 현황 표시
- 각 줄 맨 앞에 관리자 번호 표시 (예: `관리자1`, `관리자2`)
- 한 주문에 여러 품목이면 줄바꿈으로 표시
- 날짜/시간은 첫 줄에만 1회 표시, 이후 줄에는 반복 표시하지 않음

### 4.5 주문 상태 버튼 전이
- 초기: `주문접수` 버튼 표시
- `주문접수` 클릭 후: `주문접수` 버튼 숨김, `제조중` 버튼 표시
- `제조중` 클릭 후: `제조중` 버튼 숨김, `제조완료` 버튼 표시
- `제조완료` 클릭 후:
  - 해당 주문은 주문현황 목록에서 제거
  - 대시보드 `제조완료` 숫자 1 증가

### 4.6 관리자 선점/권한 규칙 (최종)
- 두 관리자 중 먼저 `주문접수`를 누른 관리자만 해당 주문을 계속 조작 가능
- 선점한 관리자만 이후 `제조중`, `제조완료` 버튼 조작 가능
- 타 관리자는 상태 버튼을 볼 수 없음(숨김)
- 추가 규칙: 타 관리자는 선점되지 않은 주문의 버튼도 숨김

## 5) 기능 요구사항 (FR)
### 5.1 주문하기
- FR-ORDER-01: 메뉴 카드는 이미지/메뉴명/가격/설명/옵션/담기 버튼을 제공해야 한다.
- FR-ORDER-02: 동일 메뉴+동일 옵션 재담기 시 장바구니 줄을 병합하고 수량만 증가해야 한다.
- FR-ORDER-03: 장바구니에서 옵션 변경은 불가해야 한다.
- FR-ORDER-04: 장바구니 항목 삭제는 각 줄 끝 `X` 버튼으로만 가능해야 한다.
- FR-ORDER-05: 장바구니 수량은 화살표 버튼으로 변경 가능해야 한다.
- FR-ORDER-06: 수량이 1일 때 감소 버튼은 비활성화되어야 한다.
- FR-ORDER-07: 주문 실패 시 장바구니를 유지해야 한다.
- FR-ORDER-08: 재고가 0이면 `담기` 버튼을 숨기고 `품절` 텍스트를 표시해야 한다.

### 5.2 관리자
- FR-ADMIN-01: 주문 상태는 `주문접수 -> 제조중 -> 제조완료` 순으로 버튼이 전이되어야 한다.
- FR-ADMIN-02: `제조완료` 클릭 시 주문은 주문현황에서 제거되고 `제조완료` 카운트가 증가해야 한다.
- FR-ADMIN-03: 주문현황 각 줄 맨 앞에 관리자 번호를 표시해야 한다.
- FR-ADMIN-04: 한 주문의 복수 품목은 줄바꿈 표시하고 날짜/시간은 첫 줄에만 표시해야 한다.
- FR-ADMIN-05: 먼저 `주문접수`를 누른 관리자만 해당 주문을 조작할 수 있어야 한다.
- FR-ADMIN-06: 타 관리자에게 상태 버튼은 숨김 처리되어야 한다.
- FR-ADMIN-07: 타 관리자에게 선점되지 않은 주문의 버튼도 숨김 처리되어야 한다.
- FR-ADMIN-08: 재고 조작 시 수량은 0 미만으로 내려가면 안 된다.

## 6) 수용 기준 (Acceptance Criteria)
- 동일 메뉴+동일 옵션을 여러 번 담으면 한 줄로 유지되고 수량만 증가한다.
- 장바구니는 화살표로 수량 증감 가능하며 수량 1에서 감소 버튼은 비활성화된다.
- 장바구니 삭제는 `X` 버튼으로만 동작한다.
- 주문 실패 시 장바구니 데이터는 유지된다.
- 재고 0 메뉴는 `담기` 대신 `품절`이 표시된다.
- 관리자 주문 버튼은 단계별로 교체되어 표시된다.
- `제조완료` 처리 시 주문은 목록에서 사라지고 대시보드 완료 수치가 증가한다.
- 복수 품목 주문은 줄바꿈되고 날짜/시간은 첫 줄에만 표시된다.
- 관리자 권한 규칙에 따라 비권한 관리자에게 버튼이 숨김 처리된다.

## 7) 백엔드 개발 PRD
### 7.1 데이터모델
- `Menus`
  - 커피 이름
  - 설명
  - 가격
  - 이미지
  - 재고 수량
- `Options`
  - 옵션 이름
  - 옵션 가격
  - 연결할 메뉴(`Menus` 참조)
- `Orders`
  - 주문 일시
  - 주문 내용(메뉴, 수량, 옵션, 금액)
  - 주문 상태(기본값: `주문 접수`)

### 7.2 데이터 스키마 기반 사용자 흐름
1. `Menus` 데이터를 조회하여 브라우저 화면에 표시한다. 이때 `재고 수량`은 관리자 화면에서만 표시한다.
2. 사용자가 앱 화면에서 커피 메뉴를 선택해 장바구니에 담으면, 선택한 정보가 장바구니에 표시된다.
3. 장바구니에서 `주문하기` 버튼 클릭 시 주문 정보를 `Orders`에 저장한다. 저장 데이터는 주문 시간과 주문 내용(메뉴, 수량, 옵션, 금액)으로 구성한다.
4. `Orders` 데이터를 관리자 화면의 `주문 현황`에 표시한다. 주문 상태 기본값은 `주문 접수`이며, `주문 접수 -> 제조 중 -> 제조 완료` 순서로 상태가 변경된다.

### 7.3 API 설계
- `주문하기` 메뉴 진입 시 데이터베이스에서 커피 메뉴 목록을 조회하여 응답한다.
- 사용자가 커피를 선택하고 `주문하기`를 실행하면 주문 정보를 데이터베이스에 저장한다.
- 주문 저장 시 주문 수량에 따라 메뉴 재고를 함께 차감/수정한다.
- 주문 ID를 전달받으면 해당 주문 상세 정보를 조회하여 응답한다.
